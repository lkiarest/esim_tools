# ESIM 管家隐私政策

最后更新：2026-06-28

`ESIM 管家` 尊重并保护你的个人信息与设备数据安全。本隐私政策用于说明应用会处理哪些信息、这些信息如何使用，以及你可以如何控制它们。

## 1. 我们处理哪些信息

应用允许你在本机保存以下信息：

- SIM / eSIM 名称
- 运营商名称
- 国家或地区
- 手机号码
- ICCID
- eSIM 激活码 / LPA 信息
- Matching ID
- 备注
- 保号提醒相关日期与周期

其中，手机号、ICCID、原始激活码和 Matching ID 会优先保存在系统安全存储中；其余普通资料保存在应用本地存储中。

## 2. 信息如何使用

这些信息仅用于以下本地功能：

- 展示和管理你的 SIM / eSIM 资料
- 生成或显示 eSIM 二维码
- 计算保号提醒时间
- 导入与导出 JSON 备份
- 在设备上发送本地通知提醒

## 3. 数据是否上传

除系统或第三方 SDK 在其自身隐私清单中声明的行为外，应用本身不提供自建账号、云同步或服务器上传逻辑，不会主动把你输入的 SIM / eSIM 资料上传到开发者服务器。

## 4. 权限使用说明

应用可能请求以下系统权限：

- 相机权限：用于扫描 eSIM 二维码
- 通知权限：用于发送本地保号提醒

iOS 平台对普通 App 的蜂窝信息访问能力有限，因此应用可能只能读取部分运营商信息，某些号码、ICCID 或停用套餐信息无法直接获取。

## 5. 第三方 SDK

应用使用 Flutter 生态中的若干系统能力或扫码相关 SDK，例如本地通知、安全存储、扫码识别及其底层依赖。相关 SDK 可能会在其 `PrivacyInfo.xcprivacy` 或官方文档中声明自身的隐私行为。

如果你通过 App Store 下载 iOS 版本，App Store 页面中的 `App Privacy` 信息将反映当前发布版本及其所包含 SDK 的隐私声明。

## 6. 数据保存与删除

- 数据默认仅保存在你的设备本地
- 你可以在应用内删除单条记录
- 你可以通过应用删除全部记录或卸载应用来移除本地数据
- 删除应用内记录不会删除系统中已安装的 SIM / eSIM

## 7. 儿童隐私

本应用不以儿童为主要目标用户，也不提供公开社交、聊天或内容分享能力。

## 8. 政策更新

如果功能或依赖发生明显变化，本隐私政策可能更新。更新后的版本会以最新日期标注。
